What Can This Toolkit Do?

The eRegistrations toolkit lets you configure, test, document, and monitor digital government services using AI agents. Instead of manually clicking through admin panels, you describe what you want and the agents handle the rest.

Configure
  • >Connect services to the central dashboard
  • >Add "My Files" panels with email delegation
  • >Set up bots, conditional logic, and workflows
Test & Verify
  • >Run end-to-end tests against production
  • >Automate form fill, submit, and status checks
  • >Correlate test results with backend logs
Document
  • >Generate user manuals for government officers
  • >Detect changes between service versions
  • >Extract live data from any configured instance
Monitor
  • >Check service health via Graylog logs
  • >Find and diagnose bot failures
  • >Trace a case through its full lifecycle

How to Get Started

  1. Install the MCP server: npx mcp-eregistrations-bpa — connects your AI agent to the BPA admin API
  2. Download skills from the Skills Catalog tab — each card has a Download .md button
  3. Place the .md files in your project:
    • .claude/skills/ — auto-loaded by Claude Code at session start
    • .claude/commands/ — invocable via /command-name
  4. Describe what you want: "Add a My Files block to this service" — the agent picks the right skill and runs it
  5. Check the Workflows tab for step-by-step guides to common end-to-end processes

Skills vs MCP Tools — When Does It Matter?

Skills are structured instructions that teach the AI agent how to accomplish a complete task. They combine multiple tool calls, domain knowledge, workarounds, and verification steps into a single workflow.

MCP tools are the 120+ individual API operations the agent uses under the hood. Think of skills as recipes and MCP tools as ingredients.

Most users only need skills and workflows. The MCP Reference tab is available for advanced users who want to understand the underlying API or build custom automations.

Config Agent Skills

2 skills

The Config Agent reads and writes to the BPA admin API. It can create bots, set up form components, configure conditional logic, and connect services together. Use these skills when you need to change how a service works.

Setup: My Files Block — Query, View & Delegate

config/setup-my-files-block.md

Adds a complete "My Files" block to any service: a button that queries all user dossiers from the Document Service API, a DataGrid table to view each file (with direct links), and a delegate action to share files via email. 16-step guide covering panel creation, bot setup, all mappings, debug scan, and publish. Includes ASCII mockup of the final result.

Config AgentReusable

What It Creates (16 steps)

  • A "My Files" panel with "Get My Files" button
  • A DataGrid with 14 columns: visible (Service Name, Business Entity, Created At, State) + hidden (File ID, Username, Metadata, etc.) + View link + Delegate button + Email field
  • Two bots: "Get my files" (query) and "Delegar file" (delegation)
  • Complete input/output mappings for both bots (using save_all)
  • Component actions wiring buttons to bots

Warnings (learned from production)

  • Do NOT pass input_param_type or service_id to bot_input_mapping_create — causes 400 errors
  • Always use save_all for DataGrid compound key mappings (_collection_ keys)
  • Output mapping orientation is reversed: source_field = form field, target_field = Mule output
  • BPA requires at least 1 input mapping per bot to publish
  • Debug scan flags _collection_ mappings as false positives — ignore those

Prerequisites

Two Mule services must be deployed: DS.DS - Query user files and DS.Share access to file. The skill checks for these first and stops if missing.

MCP Tools Used (30+ calls)

muleservice_list, bot_create (x2), form_component_add (x15), componentaction_save (x2), bot_input_mapping_create, bot_input_mapping_save_all, bot_output_mapping_save_all (x2), debug_scan, service_publish

Connects a new service to the central dashboard (Bitacora) where all dossiers/cases can be viewed in one place. An 8-step workflow that creates the necessary bots, determinants, and UI components, with verification at each step. Proven across 18 services.

The Manual Agent is read-only — it extracts data from BPA instances but never modifies them. It generates documentation, detects changes between versions, and provides structured data for other agents. Use it when you need to understand or document a service without touching its configuration.

Change Detection Between Service Versions

manual/.claude/skills/change-detection.md

Detects what changed in a service between two points in time, or compares the same service across instances (e.g., Cuba vs Jamaica). Generates structured change reports: Added / Removed / Modified / Impact.

Manual AgentReusable

MCP Tools Used

Testing Agent Skills

2 skills

The Testing Agent generates and runs end-to-end (E2E) tests using Playwright, a browser automation framework. E2E tests simulate a real user filling out forms, clicking buttons, and submitting applications — against the actual production system. Use these when you need to verify that a service works correctly from the applicant's perspective.

Observer Agent Skills

4 skills

The Observer Agent connects to Graylog, a log management platform that collects all backend activity from the eRegistrations servers. Every bot execution, API call, and error is logged. Use these skills to diagnose problems, verify that backend processes ran correctly, and monitor service health without looking at code.

Bot Failure Report

observer/.claude/skills/bot-failure-report.md

Finds all bot failures across all services. Groups them by bot name and service, detects whether failures are intermittent or consistent, and generates actionable recommendations. Start here when something seems broken.

Observer AgentReusableUses mcp-graylog

MCP Tools Used (mcp-graylog)

search_logs, get_error_logs

Service Health Check

observer/.claude/skills/service-health-check.md

Quick health assessment for any service: log volume, error rate, failed bots, empty payloads, and active users. Returns a clear HEALTHY / WARNING / CRITICAL status.

Observer AgentReusableUses mcp-graylog

MCP Tools Used (mcp-graylog)

search_logs, get_error_logs, search_stream_logs

Status Criteria

  • HEALTHY: 0 errors, all bots succeeding
  • WARNING: <5% error rate, some bot issues
  • CRITICAL: >5% error rate, systematic failures

Trace Dossier Lifecycle

observer/.claude/skills/trace-dossier.md

Follow a specific case/dossier from creation to current state through Graylog logs. Shows the full bot execution chain with timestamps and highlights any errors along the way.

Observer AgentReusableUses mcp-graylog

MCP Tools Used (mcp-graylog)

search_logs (with dossier number in message/outputData fields)

Correlate E2E Tests with Backend Logs

observer/.claude/skills/correlate-test-logs.md

After running an E2E test, finds the corresponding backend logs in Graylog (by time window, user, and service). Verdict: MATCH or MISMATCH between expected and actual bot execution sequence.

Observer AgentReusableUses mcp-graylog

search_logs (Graylog), bot_list + service_get (BPA — to correlate IDs with names)

Country skills encode this knowledge so the agent doesn't have to rediscover it every time. They can be adapted for new countries by replacing IDs and field keys.

Adapting for Another Country

Cuba E2E Service Test — Form Fill & Submit

countries/cuba/skills/e2e-service-test/SKILL.md

CubaTestingAdaptable

Key Differentiator

Cuba Choices.js dropdowns start CLOSED (click wrapper first), form.onChange() CRASHES the page, and submit detection is URL redirect + "Pendiente" text.

Jamaica SEZ — Form Fill & Submit Automation

countries/jamaica/skills/form-fill-submit/SKILL.md

Complete front-office automation for Jamaica's "Establish a new zone": 7-tab form, 30+ file uploads, DOM browse map, hidden required fields, and all platform patterns documented.

JamaicaTestingAdaptable

Reusable Patterns

Documents 5 platform patterns applicable to ALL instances: Choices.js search-based dropdowns, a.browse filechooser uploads, Form.io JS API for hidden fields, CSS unhiding for tab-panes, and JS click fallback for off-screen elements.
