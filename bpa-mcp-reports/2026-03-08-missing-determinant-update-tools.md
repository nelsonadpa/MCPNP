# BPA MCP Issue: Missing update tools for non-text determinant types

**Date:** 2026-03-08
**Reporter:** Nelson Perez (via Claude)
**Severity:** medium

## Environment

- **MCP server version:** 0.22.1 (latest: 0.22.1)
- **Instance:** BPA-jamaica (https://bpa.jamaica.eregistrations.org)
- **Service ID:** `0d8ca0c6-2a74-4d10-8c64-0b4e4a186adc`

## Summary

Only `textdeterminant_update` exists. There are no update tools for grid, boolean, radio, numeric, date, select, or classification determinants. Attempting to use `textdeterminant_update` on a grid determinant causes a 500 server error due to a Java class cast failure on the BPA backend.

## Steps to Reproduce

1. Identify a grid determinant (e.g., `c44ead04-75fe-48a4-838d-ac0488812763`, name: "ARC consolidated timeline = other (generated by copying)")
2. Call `textdeterminant_update` with `name="New name"` and `operator="EQUAL"`
3. Observe 500 error

## Actual Behavior

```
Tool: textdeterminant_update(
  service_id="0d8ca0c6-2a74-4d10-8c64-0b4e4a186adc",
  determinant_id="c44ead04-75fe-48a4-838d-ac0488812763",
  name="ARC → Board: condiciones consolidadas tienen filas",
  operator="EQUAL"
)

Response: BPA server error (500):
"class org.unctad.ereg.bpa.model.determinant.TextDeterminant cannot be
cast to class org.unctad.ereg.bpa.model.determinant.GridDeterminant"
```

Without `operator`, the tool also fails:
```
Response: "operator - must not be null"
```

## Expected Behavior

A `griddeterminant_update` tool (and equivalents for other types) should allow renaming determinants at minimum. The BPA web UI supports renaming any determinant type directly.

## Workaround Considered

Delete the grid determinant and recreate with `griddeterminant_create` using the new name. **Not viable** because deletion removes all connected effects/behaviours, which would then need to be manually re-wired — high risk of breaking service logic in production.

## Analysis

The MCP server has `*_create` tools for all determinant types but only `textdeterminant_update` for updates. The BPA REST API likely has a generic PUT endpoint for determinants (or type-specific ones) that the MCP server hasn't exposed for non-text types.

The `textdeterminant_update` tool hits the text-specific BPA endpoint, which performs a Java class cast to `TextDeterminant`. When the actual object is a `GridDeterminant`, the cast fails with a 500.

### Likely affected files

- `src/mcp_eregistrations_bpa/tools/determinant_tools.py` — needs new update functions for each determinant type, or a generic update that handles the common case (rename)

## Suggested Fix

**Option A (minimal):** Add a `determinant_rename` tool that only updates the `name` field, type-agnostic. This covers the most common use case (cleanup of "generated by copying" names, standardization).

**Option B (complete):** Add `griddeterminant_update`, `booleandeterminant_update`, `radiodeterminant_update`, `numericdeterminant_update`, `datedeterminant_update`, `selectdeterminant_update`, `classificationdeterminant_update` — mirroring the existing `*_create` tools.

**Option A is strongly preferred** — renaming is the primary need, and a single generic tool is easier to maintain.

## Impact

10 determinants in the Jamaica SEZ service ("Establish a New Zone - March") need renaming from auto-generated names to human-readable ones. Currently blocked — must be done manually via BPA web UI.
